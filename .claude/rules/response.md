# 응답 규칙

## 소통
- 한국어로 소통
- 코드 주석은 한국어 + 영어 병기

## 주석 작성 원칙 (공통)
- 주석은 최소화하고 꼭 필요한 부분에만 작성
- 코드가 말해주는 **What**은 주석 금지, 코드로 알 수 없는 **Why**만 작성
- 숨은 제약·비자명한 불변조건·특정 버그 우회·놀라운 동작일 때만 추가
- 제거해도 미래 독자가 혼란스럽지 않다면 쓰지 말 것

## 구현 전 원칙
- 규모가 Medium 이상이면 설계 플랜 먼저 제시 후 승인 대기
- 리스크가 있으면 반드시 먼저 언급
- 불확실하면 가정하지 말고 질문
- 대안이 있으면 2가지 이상 제시

## 다이어그램
- 모든 흐름도·다이어그램은 Mermaid 형식 사용 (ASCII 아트 금지)
- **Mermaid 버전: 8.8.9** — 플러그인 호환 버전 고정
  - `graph TD/LR` 사용 (`flowchart` 키워드 사용 금지)
  - 사용 가능: `-->`, `---`, `-.->`, `==>`, `subgraph`, `classDef`

## 금지 사항
- 확인 없이 파일 삭제 금지
- 확인 없이 대규모 리팩토링 금지
- 코드 변경 후 테스트 방법 안내 생략 금지
- 성능에 영향을 주는 변경 시 반드시 명시
