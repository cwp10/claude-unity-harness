```
## 설계 플랜: [기능명]
규모: Small

### 변경 위치
Scripts/Game/Player/PlayerController.cs (예시)
방식: 기존 파일에 메서드 추가

### 추가할 내용
(메서드 시그니처)

### 구현 포인트
1. ...
2. ...

### 주의사항
- ...
```

## Medium 규모 플랜

```
## 설계 플랜: [시스템명]
규모: Medium

### 기존 코드베이스 파악 결과
- 상태관리 방식: (파악한 방식) → 동일 적용
- 이벤트/통신 방식: (파악한 방식) → 동일 적용
- 참고할 유사 시스템: (있으면 명시)

### 구조 다이어그램 (Mermaid)
classDiagram

### 파일 구성
Scripts/Game/[시스템명]/
  ├── CoreClass.cs       ← 역할
  └── SubClass.cs        ← 역할

### 구현 로드맵
1단계 - 데이터 구조 (예상 Xh)
2단계 - 핵심 로직 (예상 Xh)
3단계 - 연동 및 테스트 (예상 Xh)

### 대안 방안
방안 A (권장): 장점 / 단점
방안 B: 장점 / 단점

### 예상 리스크
- 리스크: 대응책
```

## Large 규모 플랜

Medium 산출물 전체 포함 + 아래 추가:

```
### 영향 범위 분석
변경 영향을 받는 기존 시스템:
- SystemA: 어떤 변경 필요
- SystemB: 어떤 변경 필요

### 마이그레이션 플랜
Step 1: 새 시스템 병렬 구축 (기존 코드 유지)
Step 2: 새 시스템으로 신규 기능 구현
Step 3: 기존 시스템 순차 이전
Step 4: 구버전 코드 제거

### 롤백 계획
각 Step 문제 발생 시 되돌리는 방법
```

## docs/architecture/ 문서 템플릿

승인 후 자동 생성할 문서:

```markdown
# [기능명] 설계

작성일: YYYY-MM-DD
최종수정: YYYY-MM-DD

## 개요
이 시스템이 하는 역할 2~3줄

## 구조 다이어그램
(Mermaid classDiagram 또는 컴포넌트 트리)

## 파일 구성
관련 파일 목록과 역할

## 핵심 설계 결정
왜 이 구조를 선택했는지 이유

## 대안 방안
검토했던 다른 방법과 선택하지 않은 이유

## 구현 로드맵
단계별 구현 순서

## 변경 이력
| 날짜 | 변경 내용 |
|------|---------|
| YYYY-MM-DD | 최초 작성 |
```
