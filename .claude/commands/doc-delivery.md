## 실행 순서

### 1. 프로젝트 정보 수집
- `.claude/CLAUDE.md` 읽기 → 프로젝트 타입 및 기본 정보

**Unity 프로젝트:**
- `ProjectSettings/ProjectSettings.asset` → 번들 ID, 버전
- `Packages/packages-lock.json` → 사용 패키지
- `Builds/` 폴더 확인 (빌드 산출물 목록)

**웹 프로젝트:**
- `package.json` → 버전, 의존성 패키지
- `.env.example` 또는 `next.config.*` → 환경 설정
- 배포 설정 파일 (vercel.json, Dockerfile 등)

**공통:**
- Git 로그 확인: `git log --oneline -20` (최근 작업 내역)

### 2. 구현 기능 목록
`.claude/CLAUDE.md` 또는 프로젝트 내 기획 문서에서 기능 목록 파악.
없으면 폴더·파일 구조에서 추론.
Unity: 씬·스크립트 구조 기반
웹: app/ 라우트·컴포넌트 구조 기반
