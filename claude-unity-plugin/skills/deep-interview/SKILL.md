요청: $ARGUMENTS

## 실행 순서

### 1단계: 기존 컨텍스트 파악

- `.claude/project-memory.json` Read (기술 스택 및 컨벤션 파악)
- `docs/architecture/` 에서 유사 기능 설계 문서 확인
- `Assets/_Project/Scripts/` 에서 유사 패턴 Grep 탐색

### 2단계: 요구사항 명확도 판단

요청이 충분히 명확하면 (기능 범위·데이터 구조·UI·연동 시스템이 모두 특정됨):
→ "요구사항이 명확합니다. /plan 을 바로 실행합니다." 출력 후 /plan 실행

요청이 모호하면 → 3단계로 진행

### 3단계: 핵심 질문 생성

다음 카테고리에서 **모호한 항목만** 선별해 3~5개 질문:

| 카테고리 | 질문 예시 |
|----------|----------|
| 범위 | "인벤토리는 몇 개 슬롯? 장비/소비품 구분 필요?" |
| 상호작용 | "어떤 시스템과 연동? (전투, 상점, 저장)" |
| 데이터 | "런타임 상태만? 아니면 저장(JSON) 필요?" |
| UI | "HUD 표시? 별도 인벤토리 화면?" |
| 예외 처리 | "꽉 찼을 때? 아이템 중복 허용?" |
| 성능 | "아이템 수량 예상치? 풀링 필요?" |

### 4단계: 질문 출력

```
## [기능명] 요구사항 명확화

다음 항목을 답해주시면 /plan 을 바로 실행합니다:

1. [질문]
2. [질문]
3. [질문]

모두 답하거나 "기본값으로 진행"이라고 하면 /plan 을 실행합니다.
```

### 5단계: 답변 수렴 후 /plan 실행

답변을 받으면:
1. 요구사항을 한 단락으로 요약 출력
2. `/plan [기능명]` 실행 — 요구사항 요약을 컨텍스트로 함께 전달
